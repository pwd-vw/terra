Layer Detail — Components & Responsibilities

1FIELD EDGE LAYER — ESP32 Sensor & Actuator Nodes
NODE-S: Soil Sensor ×3
  • ESP32 (WROOM-32)
  • Capacitive Soil Moisture
  • DS18B20 Soil Temp
  • EC Sensor (optional)
  • Solar panel + LiPo
  • MQTT publish ทุก 5 นาที
NODE-W: Weather Station ×1
  • ESP32 DevKit
  • DHT22 / SHT31 Temp+RH
  • Rain Gauge (tipping)
  • BH1750 Light Sensor
  • Anemometer (wind)
  • ติดตั้งศูนย์กลางสวน
NODE-P: Pump Controller ×1
  • ESP32 + SSR Relay Module
  • ควบคุมปั๊มบาดาล
  • YF-S201 Flow Meter
  • Pressure sensor
  • Manual override switch
  • Alert เมื่อ flow ผิดปกติ
NODE-V: Zone Valves ×1
  • ESP32 + 4-ch Relay
  • Solenoid valve 12V ×4
  • แบ่งโซนน้ำหยด 4 โซน
  • รับ CMD จาก RPi
  • Report status ทุก cycle
IP Camera ×3–4
  • Reolink/Dahua 2MP+
  • RTSP stream → RPi
  • Night vision IR
  • กระจาย: ทางเข้า, กลางสวน, แปลงต้น
Drip Irrigation System
  • เมนท่อ PE 32mm
  • หัวน้ำหยด 2 L/hr ทุกต้น
  • Filter + Pressure Reg
  • แบ่ง 4 โซน (ควบคุม NODE-V)
2GATEWAY LAYER — Raspberry Pi Local Hub
Raspberry Pi 4 (4GB)
  • Ubuntu Server 22.04
  • SSD 128GB (USB3)
  • UPS HAT (backup power)
  • ติดตั้งในกล่องกันน้ำ + พัดลม
Mosquitto MQTT Broker
  • รับข้อมูลจาก ESP32 ทุก node
  • QoS 1 — guaranteed delivery
  • Topic structure: farm/+/+
  • TLS auth (username/pass)
Node-RED Engine
  • Automation rule engine
  • Flow: sensor → logic → action
  • เปิด/ปิดปั๊มตามเงื่อนไข
  • Schedule + threshold trigger
TimescaleDB (PostgreSQL)
  • Time-series sensor data
  • Local 30-day buffer
  • Sync ขึ้น CF D1 ทุกชั่วโมง
  • AI feature pipeline source
Frigate NVR
  • RTSP → H.264 record
  • Object detection (YOLO)
  • Motion alert → LINE
  • Snapshot → CF R2
Cloudflare Tunnel
  • cloudflared service
  • Expose API → internet
  • ไม่ต้องมี Public IP
  • Zero-Trust access control
3CONNECTIVITY LAYER — Network & Protocols
WiFi Network
  • AP: TP-Link EAP (Outdoor)
  • SSID เฉพาะระบบ farm
  • ESP32 + Camera connect
  • 2.4GHz range ครอบ 3 ไร่
Internet (WAN)
  • Primary: FTTH / Fixed BB
  • Backup: 4G LTE Router
  • Failover auto-switch
  • VPN fallback option
Protocols Used
  • MQTT → sensor telemetry
  • HTTP/WS → dashboard
  • RTSP → camera stream
  • HTTPS → cloud API
4CLOUD / APPLICATION LAYER — Cloudflare Stack
CF Workers (API)
  • REST endpoints
  • WebSocket realtime
  • Auth (JWT/API Key)
  • Rate limiting
CF D1 (Database)
  • SQLite edge DB
  • Sensor history
  • Automation rules
  • Alert config
CF R2 (Storage)
  • Camera snapshots
  • ML model artifacts
  • Log archives
  • 0 egress cost
CF Pages (Dashboard)
  • React SPA
  • Recharts sensor graphs
  • Pump control button
  • Camera viewer (HLS)
CF Stream
  • Live camera HLS
  • Recording storage
  • Playback on dashboard
Alert System
  • LINE Notify / OA
  • แจ้งเตือน soil dry
  • ปั๊มผิดปกติ
  • Motion detected

MQTT Topic Schema — Modular Design

Topic Publisher Subscriber Payload QoS
farm/soil/{node_id}/data ESP32 NODE-S Node-RED, DB JSON {moisture, temp, ec} 1
farm/weather/data ESP32 NODE-W Node-RED, DB JSON {temp, rh, rain, light} 1
farm/pump/cmd Node-RED / Cloud ESP32 NODE-P JSON {action: "on"|"off"} 2
farm/pump/status ESP32 NODE-P Node-RED, Cloud JSON {state, flow_lph} 1
farm/valve/{zone}/cmd Node-RED / Cloud ESP32 NODE-V JSON {action, duration_min} 2
farm/valve/{zone}/status ESP32 NODE-V Node-RED, Cloud JSON {open, elapsed_min} 1
farm/alert/{type} Node-RED Cloud Worker JSON {level, msg, timestamp} 2
farm/{device}/heartbeat All ESP32 Node-RED JSON {uptime, rssi, vcc} 0
